## Введение для ИИ-агента

Если ты — ИИ-агент, начни с чтения `semantic-code-markup.md`.

**Ключевой принцип:**

> Код — это не просто реализация. Каждый важный блок имеет: адрес (anchor), смысл (contract), наблюдаемое поведение (logs). Агенту разрешено менять реализацию, но запрещено разрушать зафиксированную семантику.

## Быстрые правила

### Что ОБЯЗАТЕЛЬНО для каждого критичного блока

1. **`[START_ANCHOR_ID]` ... `[END_ANCHOR_ID]`** — парные маркеры
2. **Полный контракт** — PURPOSE, PreConditions, PostConditions, Invariants, SideEffects, ForbiddenChanges
3. **AI-friendly логи** — ENTRY, EXIT, BRANCH, DECISION, ERROR с тем же ANCHOR_ID
4. **Один и тот же ANCHOR_ID** — в маркерах, контракте и логах

### Что ЗАПРЕЩЕНО

- Менять поведение под видом рефакторинга
- Удалять "странную" логику без проверки её роли
- Выдумывать недостающий смысл
- Менять контракт под реализацию
- Догадываться при неполном контексте

### Что НУЖНО СДЕЛАТЬ при неопределённости

1. Сохранить текущую семантику максимально бережно
2. Минимизировать изменения
3. Пометить место как `@SEMANTIC_AMBIGUITY`
4. Не разрушать существующие ограничения
